The git branch command in Git is used for various branch-related operations. Here are some common use cases:

List Branches:
To list all the branches in your repository, you can use:
bash
Copy code
git branch
The currently active branch is marked with an asterisk (*).
Create a New Branch:
To create a new branch, you can use:
bash
Copy code
git branch <branch_name>
This creates a new branch but does not switch to it. You can switch to the new branch using git checkout or git switch.
bash
Copy code
git checkout <branch_name>
# or
git switch <branch_name>
Alternatively, you can create and switch to a new branch in one command:
bash
Copy code
git checkout -b <branch_name>
# or
git switch -c <branch_name>
Delete a Branch:
To delete a branch, use:
bash
Copy code
git branch -d <branch_name>
This deletes the specified branch if its changes are already merged into the current branch. If you want to force-delete a branch, even if its changes are not merged, you can use -D:
bash
Copy code
git branch -D <branch_name>
Rename a Branch:
To rename a branch, you can use the -m option:
bash
Copy code
git branch -m <new_branch_name>
Switch Between Branches:
To switch to a different branch, you can use:
bash
Copy code
git checkout <branch_name>
# or
git switch <branch_name>
In Git version 2.23 and later, you can use git switch for a more user-friendly experience when switching branches.
Show Current Branch:
To display the name of the currently active branch, you can use:
bash
Copy code
git branch --show-current
These are some of the basic use cases of the git branch command. The command is versatile and has additional options for more advanced use cases. You can check the Git documentation or use git branch --help for more details and options.
